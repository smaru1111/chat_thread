# 07. Experiment Flow - 実験フロー（Google Forms運用）

## 📋 目的
本資料は、AIチャットアプリにおける **「線形チャット」** と **「スレッド型チャット（1階層）」** のUXを比較評価するための、Google Forms上の実験フローと設問仕様をまとめます。

- **所要時間**：10〜15分
- **前提**：Google または GitHub でログインして利用する
- **実験URL（ベース）**：`https://chat-thread.vercel.app/`

評価設計の概要は `01_research.md` を参照してください。

---

## 🔗 会話ログ分析のための紐づけ方（会話IDコピペ運用）
本実験では、アンケート回答とアプリ内の会話内容（ログ）を紐づけて分析します。  
そのため、各条件のタスク完了後に **「会話ID」** をアプリからコピーして、Google Formsへ貼り付けてもらいます。

### 参加者への注意（フォーム/依頼文で明記推奨）
- **個人情報は入力しない**：氏名、住所、連絡先、学籍番号などの個人情報は会話に含めないよう依頼する
- **条件ごとに会話を分ける**：線形とスレッドは **別の会話（別ID）** で実施してもらう（混在防止）
- **会話IDは必須**：会話IDが未入力だと会話ログと紐づけできず、分析対象から外れる可能性がある

### 実験者向けメモ
- フォームでは **「線形の会話ID」** と **「スレッドの会話ID」** を別設問で回収する
- 回収した会話IDをキーとして、DB/ログから会話内容を抽出して分析する

### セクション3：線形チャットの実施（タスク + 5段階評価）
#### 実施指示（フォーム記載例）
> ここから線形チャットの実験を実施します。  
> アプリを開き、線形モードで以下のタスクをこなし、完了したら **会話IDをコピーして** アンケートに答えてください。
>
> お題：週末の過ごし方の提案  
> やること：  
> - まず「やりたいこと」を相談してください（例：休息/運動/勉強/外出など）  
> - その後、適宜相談しながら「最終案」を1つにまとめてください

#### 実施リンク（推奨）
線形モードに迷わず入れるよう、フォーム内には以下のリンクを提示します。
- `https://chat-thread.vercel.app/?mode=linear`

#### 会話IDの貼り付け（必須）
タスク完了後、アプリの会話画面に表示される **「会話IDをコピー」** から会話IDをコピーし、以下に貼り付けてください。  
（コピーできない場合は、表示されている会話IDをそのまま貼り付け/手入力してください）

- 線形チャットの会話ID（必須）

#### 5段階評価（必須）
各項目を **1（低い）〜5（高い）** で回答。

- 使いやすさ：操作が直感的で、迷わず利用できた。
- 会話の分かりやすさ：今どの話題（文脈）を扱っているか理解しやすかった
- 話題への復帰のしやすさ：前の話題に戻って続きから会話しやすかった
- 混線のしにくさ：別の話題が混ざって混乱することが少なかった
- 満足度：全体として満足できる体験だった

- 回答が揃ったら、各項目の平均（必要なら標準偏差）を線形/スレッドで並べて表にする
